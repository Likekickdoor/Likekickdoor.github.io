操作系统知识点

发表于 2019-05-02 | 分类于 OperatingSystem

基本概念

  1. 操作系统（OS）：是一组控制和管理计算机硬件和软件资源、合理地对各类作业进行调度，以及方便用户的程序的集合。

    • 作用：
      1. 用户与计算机之间的接口
      2. 计算机系统资源的管理者（计算机资源包括：运算器、控制器、存储器、输入输出设备）
      3. 实现计算机资源的抽象
    • 功能：
      1. 处理机管理：进程控制、进程同步、进程通信、进程调度
      2. 存储管理：内存分配、内存保护、地址映射、内存扩充
      3. 设备管理：缓冲区管理、设备分配、设备处理、虚拟设备管理
      4. 文件管理：文件存储空间管理、目录管理、文件读写管理、文件保护、文件系统的安全性
      5. 用户接口：命令接口、程序接口、图形接口
    • 基本特征：
      1. 并发性，是指在一段时间内多道程序同时在计算机内运行
      2. 共享性，多道程序多用户作业共享有限的计算机系统资源
        • 互斥共享：指该类资源的分配必须以作业（或进程）为单位，在一个作业（或进程）没有运行完成前，另一个作业不得使用该资源
        • 同时共享：多个作业（或进程）“同时”（一段时间内，替换轮换）共享资源。某和时刻只有一个作业运行。
      3. 虚拟性，指操作系统使用某种技术，将物理上的一个资源或设备编程逻辑上的多个资源或设备。
      4. 异步性，进程（作业）以人们不可预知的速度向前推进

  2. 中断： 指CPU在执行一个程序时，对系统发生的某个事件（程序自身或外界的原因引起的）作出的一种反应。

    • 特点：随机性、可恢复性（因有中断寄存器和程序状态寄存器）、自动性
    • 作业：
      1. 实现CPU和I/O设备并行工作
      2. 实现硬件故障处理
      3. 实现人-机通信
      4. 实现多道程序和分时操作
      5. 实时处理
      6. 实现应用程序和操作系统联系
      7. 实现多处理机系统中处理机之间联系
    • 中断屏蔽：PSW（即程序状态字（有些教材也叫程序状态寄存器），ProgramStatusWord）中的中断屏蔽位与一些中断时间对应，当某一位有屏蔽标志时，表示屏蔽。
    • 地位：中断是多道程序并发执行的推动力。操作系统是由中断驱动的。
    • 向量中断：当中断发生时，由中断源引起CPU进入中断服务程序的中断过程。
    • 中断向量：在向量中断中，对每一个中断类型都设置一个中断向量，其中包括“中断服务程序的入口地址”和“处理器的PSW”。

进程和线程

进程：

  1. 程序顺序执行的特征：

    • 顺序性
    • 封闭性
    • 可再现性

  2. 程序并发执行的特征：

    • 间断性（异步性，有“执行–暂停–执行特征”）
    • 失去封闭性
    • 失去不可再现性

  3. 资源分配和独立运行 的基本单位都是 进程 。进程是操作系统中的核心概念之一。

    1. 定义：
      • 进程是程序的一次进行。
      • 进程是可以和别的计算并发执行的计算。
      • 进程是定义在一个数据结构上，并能够在其上进行操作的一个程序。
      • 进程是在程序在一个数据集合上运行的过程，它是操作系统进行资源分配和调度的独立单位。
    2. 特征：
      • 具有动态性。
      • 可包含多个程序。

  4. 进程状态：

    1. 两种状态模型：运行态 和 非运行态

    2. 五状态模型：(实际七状态模型)

      引入挂起的进程状态转换模型：
          增加了“挂起”的概念，当内存中没有就绪的进程时，系统将内存中处于阻塞的进程换出到外存disk（虚拟内存）中的挂起队列（阻塞挂起态）

  5. 在进程从一种状态转变到另一种状态的过程中，系统发挥“指挥调度”作用；

    • PCB（进程控制块）：有一种数据结构描述每一个进程的相关信息，以便操作系统对其进行跟踪和调派，该数据结构便是PCB
    • 定义：进程控制块PCB是操作系统用于记录进程状态和相关信息，控制进程运行的数据结构，是进程的唯一标识符。
    • 其主要包含信息：
      1. 进程标识符；
      2. 进程状态；
      3. 调度信息；
      4. 程序计数器；
      5. 上下文数据；
      6. 内存指针；
      7. I/O状态信息；

  6. 原语是原子操作，是不可分割的基本单元，在执行过程中不会被打断。

    • 阻塞原语 <—> 唤醒原语
    • 挂起原语 <—> 激活原语

线程：

一个与进程类似的单独实体来完成调度，该实体是线程。

  1. 概念：通常把调度和分派的基本单位成为线程。

  2. 特征：

    • 并发性；
    • 共享性；
    • 动态性；
    • 结构性；

  3. 状态：（挂起对线程没有意义）

    • 运行态；
    • 就绪态；
    • 阻塞态；

  4. 线程实现：

    • 用户级线程；
    • 内核级线程；
    • 组合线程；

  5. 线程模型：用户线程与内核线程之间存在的一种关系

    • 多对一模型；
    • 一对一模型；
    • 多对多模型；

互斥与同步

  1. 并发原理：

    • 相关术语：临界区、竞争、同步、互斥、死锁、饥饿（概念需要自己查询）
    • 两种制约关系：
      • 直接相互制约关系（同步）
      • 间接互相制约关系（互斥）
    • 临界资源和临界区：
      • 多个并发执行的进程在产生竞争时用同一个资源会产生冲突。
      • 临界资源：指如，打印机、磁盘机等硬件资源；
      • 临界区：使用临界资源的程序代码称为临界区；
      • 实现互斥进入临界区 –即可–> 实现互斥访问临界资源
      • 互斥产生两个问题：死锁、饥饿
      • 多个进程进入共享临界区必须遵循的原则：空闲让进、忙则等待、有限等待、让全等待

  2. 硬件同步：临界区问题都需要一个简单的锁工具，临界区通过锁的防护可以避免竞争。

    • 关中断
    • TestAndSet指令（该指令是一种硬件指令实现互斥且是原语）
    • Swap指令

调度与死锁

调度：处理器的调度正是处理（处理器和内存资源）调度和分配的相关的工作。

  1. 基本概念：
    处理器分为三级，高级调度、中级调度、初级调度

    • 高级调度：又称为作业调度或长调度，调度对象是作业。
    • 低级调度：又称为进程调度或短调度，调度对象是进程。
    • 中级调度：介于高级调度和低级调度之间，该调度根据进程状态决定辅存和主存之间的进程对换。

  2. 作业：是一种比程序更为广泛的概念，不仅包括通常的程序和数据，还配有说明书来对作业的运行进行控制。

  3. 调度原则：

    1. 周转时间：是指作业提交给系统开始，到作业完成的这段时间间隔。

      其意义是，评价批处理系统的性能、选择作业调度方式与算法的重要准则之一

      周转时间 = 后备队列等待时间 + 就绪队列等待时间* + CPU执行时间* + 等待I/O完成时间* （注：*表示代表可能发生多次）

      平均周转时间：其意义，用于衡量不同作业调度算法对同一作业流的调度性能。

      平均带权周转时间: 其意义，用于衡量同一作业调度算法对不同作业流的调度性能。

    2. 响应时间：用于评价分时系统的性能，这里选择分时系统中进程调度算法的重要原则之一。
      响应时间 = 请求信息传送到处理机的时间 + 处理机处理信息的时间 + 响应信息返回终端时间

    3. 截止时间：评价实时系统性能的重要指标，是选择实时调度算法的准则。

    4. 优先权

    5. 处理机利用率

    6. 公平性

    7. 吞吐量：指在单位时间内系统所完成作业数，与批处理作业的平均长度密切有关。

  4. 调度算法：3个，FCFS、SJF、优先级调度算法、时间片轮转调度算法

    1. 先来先服务算法（FCFS）：用于短调度、长调度；有利于长作业（长进程）；其平均周转时间和平均带权周转时间受作业到达顺序和调度顺序影响。

    2. 优先级调度算法：按时限设定好的进程优先级选取级别最高者；

    • 抢占式（出现优先级更高者的，当前进程立即停止，能更好的满足紧迫作业，结合实时性高的系统）
    • 非抢占式（出现优先级更高者的，当前进程继续运行至结束或因等待时间而阻塞，然后更高级别者用处理机，多用于批处理系统）
    1. 时间片轮转调度算法：将所有就绪进程按先来先服务原则排列成一个队列，每次调度时，首进程为其分配时间片，用完后中断，处理机执行下一个进程，该进程置于队尾。
      时间片大小对系统性能有很大影响。
时间片短 时间片较长
利于短作业 退化为先来先服务调度算法
开销大 无法满足用户需求

死锁：由于临界资源的出现，以及允许进程并发执行，就会出现进程申请资源而被永远阻塞的情况，即为死锁

  1. 资源：
    可剥夺性资源，如内存、处理机；
    非剥夺性资源，如进程得到在使用完前，不能被别程序夺走，如打印机、磁带机；

  2. 死锁产生的原因：资源竞争不当以及进程的推进顺序不当有关。

    • 资源竞争：临界资源不能同时满足多个进程请求。
    • 进程推进顺序不当：进程本身的异步性决定了在其请求和释放资源时，选择时机不当可能会发生。

  3. 死锁产生的必要条件：互斥、请求和保持、不可剥夺、循环等待

  4. 死锁的预防：

    • 破坏“请求和保持”条件；
    • 破坏“不可剥夺”条件；
    • 破坏“环路等待”条件；

  5. 死锁的避免：通过合理的资源分配确保不会出现循环等待的条件，从而避免死锁发生且更好支持并发资源合理使用。如，银行家算法。

  6. 死锁检测和恢复

    • 死锁检验：死锁定理、安全性算法 都是进行死锁检测的常见算法。

    • 死锁恢复：一旦检验到系统，就需要采用某种策略将其从死锁状态中恢复出来。可以从 剥夺资源 和 撤销进程 两个方面进行死锁恢复。

      恢复方法：

    1. 取消所有死锁进程；
    2. 让每个死锁进程回退到上述某些安全性检查的时间点前，并重新启动所有进程；
    3. 连续取消死锁进程直到不再存在死锁为止；
    4. 连续抢占资源直到不再存在死锁为止；

内存管理

现代计算机都是以存储器为中心进行设计的，因此存储器管理成为操作系统的主要任务之一。

存储器层次结构：

  • 第一层：寄存器、高速缓冲存储器
  • 第二层：内存、磁盘缓存
  • 第三层：外存、可移动磁盘介质

存储器管理的目的和任务：

  • 目的：要求操作系统的存储管理机构必须尽可能 1、方便用户和提高内存的使用效率，2、使内存在成本、速度和规模之间获得较好的权衡；
  • 任务：内存分配、地址映射（将程序逻辑地址转化为计算机认识的物理地址）、内存共享与保护、内存扩充（得到比物理空间更大的虚拟逻辑空间）
